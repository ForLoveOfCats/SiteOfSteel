46 ## Building
47 
48 ### Prerequisites
49 
50 * A functioning installation of Godot Mono 3.1.1 Stable
51  * Decently recent versions of both MSBuild and Nuget must be installed (Windows users make sure
52  to install [this](https://www.mono-project.com/download/stable/#download-win) and
53  [this](https://www.microsoft.com/en-us/download/details.aspx?id=56119), Ubuntu users make
54  sure to add [this PPA](https://www.mono-project.com/download/stable/#download-lin), update,
55  then install the mono packages as for Arch/Manjaro users make sure to install `msbuild-stable` from the AUR)
56 
57 
58 ### Performing the Build
59 
60 * Clone the git repo (or download as zip and extract)
61 * Cd into the project root and run `nuget restore SkyOfSteel.sln` (This will pull in Newtonsoft.Json and Roslyn)
62 * Open the Godot editor to the project and let it reimport all the assets then hit "Play" near the upper right corner (play icon)
63  * The editor will proceed to build and launch the project

67 ## Gameplay
68 
69 SkyOfSteel is a sandbox building game set in the sky. Envisioned to be
70 a game where one can build intricate factories and sprawling supply
71 chains while engaging in a PVP turf war with your fellow server
72 inhabitants. Enjoy highly fluid movement while destroying others with
73 highly powered weaponry. Fight for technological superiority as you
74 race through a deep tech tree constantly edging out your
75 opponents. When you tire from the vanilla game, the custom gamemode
76 and modding support adds even more depth and content.
77 
78 Currently we are remarkably far along in achieving this goal. However
79 we are admittedly not there yet. At the moment we have a fully
80 functional building system, chunk based world save and loading, highly
81 stable multiplayer, Quake/Source engine style air strafing, automatic
82 bunny hopping, wall kicking, and a rudimentary yet extremely flexible
83 system for adding user created gamemodes.

87 ## Modding
88 
89 SkyOfSteel has a gamemode modding system utilizing [Roslyn](https://github.com/dotnet/roslyn "Roslyn Github Page") to provide runtime
90 C# compilation. The intention is to allow for fully custom gamemodes to be developed with full access to internal game APIs both
91 server side *and* client side. In addition there is a full ingame REPL console.
92 
93 The gamemode API has a concept of game events. Whenever something occurs ingame it is
94 processed as an "event" which can be filtered and discarded by gamemode scripts programmatically.
95 This allows for a great deal of flexibility before even touching the normal API.
96 
97 Read more here: https://github.com/ForLoveOfCats/ModsOfSteel

101 ## Player Input Handling
102 
103 All player input is routed through the binding system which depending on the input will call
104 a console command passing in a `float` ranging between 0 and 1 (up to infinity in some cases
105 such as mouse movement). Key input pass in a 1 for keydown and a 0 for keyup while analog
106 inputs such as mouse motion pass in a 0 for no movement on the specified axis and a number
107 greater than 0 for motion on that axis (the number is the amount of motion).
108 
109 **Note:** This is not restricted to built in movement and player control console commands.
110 Any function declared in the console REPL can be bound to any key or input.

114 ## Internal Structure
115 
116 SkyOfSteel is built up of a number of singleton "libraries". Each one of these libraries deals
117 with a portion of the game's actions and has a number of public static methods in order to be
118 called from other libraries and classes.
119 
120 Beyond these "libraries" the project directory is split up into pretty self explanatory folders
121 which are almost like "modules" in that they (under most conditions) contain everything related
122 to the folder name (ex: the `UI` folder has alls GUI scenes, script, and textures). Note that
123 these folders are not self contained but instead all source files are pretty much entwined though
124 the folder names are a good indication of where to look when trying to find the implementation
125 of a specific feature.
